# 감도 (GAMDO) — 7일 프로토타입 디자인 참고서 v1.0

**작성일:** 2026-07-23  
**목적:** UX 분석에서 얻은 원칙을 감도(GAMDO)의 확정 요구사항과 2인·7일 개발 범위에 맞게 화면 설계로 번역한다.

> 이 문서는 새 기능 목록이 아니다. PRD와 기능명세서의 P0 흐름을 실제 화면으로 일관되게 구현하기 위한 디자인 기준이다. 외부 앱의 고유한 색·아이콘·카피·화면을 복제하지 않는다.

## 0. 적용 기준과 문서 우선순위

### 제품명과 용어

- 사용자에게 보이는 서비스명은 **감도(GAMDO)** 로 통일한다.
- `MOODFRAME`은 기존 제안서와 작업 맥락을 위한 **내부 코드명**으로만 사용한다.
- UI에는 가능한 한 쉬운 한국어를 쓴다. 예: `Mood`가 아니라 `무드`, `Rescue`가 아니라 `사진 살리기`.

### 문서 우선순위

우선순위·컷라인은 아래 순서로 해석한다.

1. `감도_GAMDO_PRD_v1.0_1.md`의 범위와 컷라인
2. `감도_GAMDO_기능명세서_v1.0_2.md`의 모듈별 완료 조건
3. `감도_GAMDO_DB스키마_v2.0.md`의 데이터·보관 정책
4. `MOODFRAME_2인_개발_로드맵.md`의 일자별 구현 순서
5. 영뽀또 UX 분석 및 레이아웃 반영 계획의 사용성 원칙

UX 분석의 앱 순위·평점·평가 수처럼 변동 가능한 스토어 수치는 이 문서에서 제품 근거로 사용하지 않는다. 외부 사례에서 취할 것은 **사진으로 고르고, 카메라 안에서 바로 실행한다**는 정보 구조뿐이다.

## 1. 디자인 한 문장

> 좋아하는 사진으로 무드를 고르고, 카메라에서 한 가지 행동만 안내받아 찍고, 같은 무드로 결과를 완성한다.

감도의 핵심은 예쁜 필터나 AI 생성 결과가 아니라, 사용자의 취향을 **촬영 시점에 작동하는 개인 촬영 디렉터**로 바꾸는 경험이다.

### 반드시 지킬 UX 원칙

1. **사진 우선:** 스타일의 설명보다 결과 이미지를 먼저 보여준다.
2. **학습과 실행을 분리하지 않음:** 스타일을 고른 직후 같은 흐름에서 카메라를 연다.
3. **카메라가 중심:** 스타일 변경·레퍼런스 선택·실시간 안내는 카메라를 떠나지 않고 수행한다.
4. **한 번에 한 가지:** 카메라 안내도, 다음 행동도, 상태 경고도 우선순위가 가장 높은 한 가지를 먼저 보여준다.
5. **원본 보호와 처리 위치의 투명성:** 로컬 보정과 서버 생성의 차이를 화면에서 구분한다.
6. **카메라 도구처럼 보이기:** 과도한 AI 장식·전문 용어·많은 설정은 피한다.

## 2. 이번 버전의 결정 사항

| 항목 | 이번 문서의 결정 | 근거·범위 |
|---|---|---|
| 최상위 구조 | `무드 / 카메라 / 사진 살리기 / 앨범` 4탭 | 홈의 3대 진입을 더 빠르게 만들기 위해 채택. 기존 M15는 다음 문서 정리 때 이 구조로 개정한다. |

| 카메라 하단 선택 | P0에서는 `무드`와 `레퍼런스`만 노출 | `구도·포즈·색감`은 선택한 무드와 레퍼런스에서 자동 결정한다. 자리만 차지하는 비활성 탭도 노출하지 않는다. |
| 온보딩 건너뛰기 | 채택 | 기본 프로필로 시작한다는 점을 명시하고, 이후 무드 탭 설정에서 다시 선택할 수 있게 한다. |
| 스타일 상세 | P0에서는 대표 사진 + 2줄 설명 + 촬영 CTA | 2×2 모자이크와 3~6장 예시는 P1. 스타일 콘텐츠 제작량을 통제한다. |
| 앨범 메타 | P0은 3열 그리드와 생성 보완 뱃지까지만 | 일치도·세부 편집 이력·스타일별 분류는 데이터가 있어도 P1 상세 화면에서 제공한다. |
| 음성 안내 | P2, 노출하지 않음 | M6-04의 음성 안내 우선순위와 일치. |
| 개인정보 문구 | 서버 전송·삭제 사실만 표현 | HTTPS 등 구현 확인 전에는 `암호화해 전송`이라고 단정하지 않는다. |

### 범위 보호 규칙

- Day 5 시점에 생성 API가 불안정하면 **생성 보완 탭과 생성 CTA를 숨기고**, 기본 보정 결과로 폴백한다.
- 객체 제거와 여백 확장이 모두 준비되지 않았다면, 고정 데모 시나리오에서 검증된 **한 가지 생성 보완만 노출**한다.
- 오버레이가 불안정하면 정적 프리셋 프레임으로 다운그레이드하고, 가이드 품질을 우선한다.
- P1/P2 기능은 탭·버튼·빈 카드로 미리 노출하지 않는다. 보이지 않는 기능은 사용자가 잃어버릴 기능이 아니다.

## 3. 정보 구조와 핵심 흐름

### P0 전역 내비게이션

```text
┌──────────────────────────────────────────────────┐
│  무드              카메라       사진 살리기   앨범 │
└──────────────────────────────────────────────────┘
```

- **무드:** 개인화 추천 스타일 3종과 즉시 촬영 진입점
- **카메라:** 선택한 무드에 맞는 실시간 촬영 가이드
- **사진 살리기:** 기존 사진의 문제 진단과 기본/생성 보완
- **앨범:** 앱에서 촬영·저장한 원본과 결과의 빠른 재진입

`결과`는 촬영과 사진 살리기 양쪽에서 열리는 **맥락 화면**이며, 하단 탭에 넣지 않는다. `내 스타일`, 설정, 개인정보 안내는 무드 탭 오른쪽 위의 시트에서 연다.

### P0 사용자 흐름

```text
온보딩: 사진 카드 5장 이상 선택
        ↓
무드: 추천 3종 중 하나 선택 ──→ 카메라
        │                              ↓
        └──── 바로 촬영 ──────────→ 오버레이 확인 · 수동 촬영
                                       ↓
                                    결과 4트랙 → 저장 → 1탭 피드백

사진 살리기: 사진 선택 → 문제 진단 → 기본/생성 보완 → 결과 4트랙 → 저장
앨범: 사진 선택 ────────────────────────────────────────────────→ 결과
```

### P1 이후에만 추가할 구조

```text
무드: 전체 스타일 탐색 · 카테고리 · 스타일 상세 · 저장 프리셋
카메라: 포즈 선택 · 수동 구도 선택 · 색감 강도 조절
앨범: 스타일별 분류 · 일치도 · 상세 편집 이력 · 즐겨찾기
```

## 4. 화면별 디자인 기준

### 4.1 온보딩 — 사진으로 취향 고르기

**목표:** 신규 사용자가 60초 안에 첫 카메라 화면까지 도달한다(M2-01).

흐름은 `소개 → 사진/카메라 권한 → 취향 카드 → 프로필 결과 → 개인정보 안내`를 유지한다. 취향 카드 단계만 이미지 중심으로 강하게 설계한다.

```text
[건너뛰기]                                      [3/5]

어떤 사진에 더 마음이 가나요?
좋아하는 사진을 5장 이상 골라주세요.

┌──────────────┐  ┌──────────────┐
│              │  │              │
│    사진 카드   │  │    사진 카드   │
│              │  │              │
└──────────────┘  └──────────────┘

[ 3장 선택됨 · 2장 더 골라주세요 ]
```

- 2열 고정 그리드, 카드 비율은 세로 사진이 살아나는 `3:4`를 기본으로 한다.
- 카드는 M2-02의 20장과 사전 계산된 특성 벡터를 사용한다. 카드의 이름·기술 수치는 처음부터 보여주지 않는다.
- 선택 상태는 이미지 위를 어둡게 덮지 않고, 대비가 충분한 테두리와 작은 체크로만 표시한다.
- `건너뛰기`를 선택하면 기본 프로필·기본 추천으로 진입한다. 건너뛰기 후에도 카메라 사용을 막지 않는다.
- 프로필 결과는 일상어 1~2문장과 추천 스타일 3장만 보여준다. 예: `밝은 자연광과 넓은 배경의 자연스러운 인물 사진을 좋아해요.`
- 개인정보 안내에는 로컬 원본 보존과 **앱 삭제 시 로컬 데이터가 사라질 수 있음**을 함께 적는다.

### 4.2 무드 — 홈과 최소 스타일 선택을 결합

**목표:** 사용자가 스타일 설명을 읽지 않고도 1~2번의 터치로 촬영을 시작한다.

```text
무드                                             [설정]
오늘은 어떤 느낌으로 찍어볼까요?

추천 무드
┌──────────────────────┐ ┌──────────────────────┐
│      대표 사진         │ │      대표 사진         │
│  자연스러운 산책        │ │  따뜻한 카페            │
│  넓은 배경 · 부드러운 색 │ │  밝고 포근하게          │
└──────────────────────┘ └──────────────────────┘

┌──────────────────────┐
│      대표 사진         │
│  밤거리 스냅            │
└──────────────────────┘

[ 바로 촬영하기 ]       [ 레퍼런스 따라 찍기 ]
```

- 추천 카드 3종은 M2-03의 개인화 결과를 사용한다. 카드가 3개 미만이면 기본 프리셋으로 채운다.
- 카드 탭은 해당 무드를 선택한 뒤 카메라로 바로 이동한다. `바로 촬영하기`는 현재 선택 무드로 이동한다.
- `레퍼런스 따라 찍기`는 사진 선택 후 카메라로 이동한다(M7-01). 무드를 추가로 고르게 하지 않는다.
- 검색, 인기 순위, 감성 카테고리, 스타일 혼합, 긴 프리뷰, 전체 3열 스타일 그리드는 P1/P2다.
- 설정 시트에는 `취향 다시 선택`, `개인화 초기화`, 개인정보 안내만 둔다. 별도 프로필 화면은 P1이다.

### 4.3 카메라 — 개인 촬영 디렉터

**목표:** 카메라 진입 후 2초 이내에 목표 구도와 한 줄 안내를 제시한다(M6-01).

```text
닫기          현재 무드 ▾                         오버레이 켜기

┌──────────────────── 카메라 프리뷰 ────────────────────┐
│                                                        │
│     목표 프레임 · 피사체 위치 마커 · 수평 가이드          │
│                                                        │
│              ← 오른쪽으로 한 걸음 이동하세요             │
│                                                        │
└────────────────────────────────────────────────────────┘

[ 무드 ] [ 레퍼런스 ]
[ 자연스러운 산책 ]  [ 따뜻한 카페 ]  [ 밤거리 스냅 ]

최근 사진                 ◯ 일치도 링 · 셔터                 전후면 전환
```

- 상단에는 닫기, 현재 무드 변경, 오버레이 표시 토글만 기본 노출한다. 플래시·비율·렌즈 같은 기기 제어는 아이콘 한 개의 보조 시트로 보낸다.
- 프리뷰에는 추천 프레임, 피사체 배치 지점, 수평선 등 **현재 안내에 필요한 오버레이만** 표시한다. 모든 감지값을 숫자로 나열하지 않는다.
- 안내 배너는 화면 하단 프리뷰 안에 1줄로 배치한다. M6-03에 따라 최소 0.8초 유지하며, 직전 안내와 충돌하는 문구를 즉시 교체하지 않는다.
- 셔터 테두리의 얇은 링이 일치도를 표시한다. 별도의 큰 점수 카드나 퍼센트 표시는 만들지 않는다.
- 기본 하단 캐러셀은 `무드`와 `레퍼런스`만 있다. `구도`, `포즈`, `색감`은 자동 해석 결과이며 별도 선택 행을 만들지 않는다.
- 레퍼런스 모드에서는 구조화 실루엣과 원본 반투명 오버레이를 전환한다(M7-03). 업로드 목적과 분석 후 삭제를 선택 직전에 고지한다.
- 수동 셔터는 항상 활성화한다. 목표 영역에 들어오면 오버레이 색만 전환한다. 자동 촬영 준비 상태·카운트다운·셔터 링은 제공하지 않는다.

#### 카메라 상태 문구

| 상태 | P0 표현 | 동작 |
|---|---|---|
| 인물 미검출 | `인물이 화면에 들어오면 구도를 맞춰드릴게요.` | 인물 기반 오버레이를 숨기고 수평·구도 프레임만 유지 |
| 감지 불안정 | 안내 배너 숨김 | 마지막 안정된 목표 프레임을 잠시 유지 |
| 저조도 | `조명이 어두워요.` | 상태 칩만 표시하고 촬영은 사용자가 판단 |
| 다인 인물 | `누구를 중심으로 찍을까요?` + 얼굴 마커 | P1 우선. P0에서는 기본 인물 1명을 선택하거나 단일 인물 시연으로 제한 |
| 네트워크 없음 | `기본 촬영과 보정은 계속 사용할 수 있어요.` | 생성 보완만 비활성 |

### 4.4 사진 살리기 — 문제를 먼저 설명하고 결과를 비교

**목표:** 기존 사진의 실패 원인을 사용자가 이해할 수 있는 말로 보여주고, 로컬 보정과 생성 보완의 처리 차이를 분명히 한다.

```text
사진 살리기

[ 사진 선택 ]

이 사진을 이렇게 다듬어볼까요?
┌────────────────────────────────┐
│ ✓ 기울어진 화면을 바로잡기       │
│ ✓ 어두운 부분을 자연스럽게 밝히기 │
│ □ 오른쪽의 방해 요소 지우기       │
└────────────────────────────────┘

[ 기본 보정 적용 ]   [ 생성 보완 보기 ]
```

- 첫 진입은 OS 사진 선택기를 사용한다. 별도 사진 분류·검색·3열 미디어 브라우저는 P1이다.
- 문제 카드는 M12-02의 진단 결과를 한 카드에 한 문제씩 간단히 표현한다. 내부 수치(예: 기울기 각도)는 전면에 노출하지 않는다.
- `기본 보정 적용`은 수평·크롭·노출·색감을 로컬에서 처리한다(M10).
- `생성 보완 보기`는 업로드 직전에 아래처럼 고지한다.

  > 클라우드 처리 필요  
  > 사진을 편집하기 위해 서버로 전송하며, 작업이 끝나면 원본은 삭제됩니다.

- 전송 암호화, 저장 기간 등은 실제 구현·운영 정책이 확정된 뒤에만 더 구체적으로 표기한다. 현재 명세 기준으로 입력 원본은 job 종료 즉시, 결과 파일은 수신 확인 후 최대 24시간 내 삭제된다.
- 얼굴 일관성을 보장하지 못한 생성 결과는 표시하지 않고, `자연스러운 보정만 적용했어요.`라는 기본 보정 폴백을 제시한다.

### 4.5 결과 — 네 가지 트랙, 하나의 비교 경험

**목표:** 사용자가 원본과 결과의 차이, 생성 보완 여부, 되돌릴 수 있다는 사실을 한 화면에서 이해한다.

```text
뒤로                                      저장 · 공유

                 [ 사진 크게 보기 ]
           길게 누르면 원본 / 놓으면 현재 결과

원본 | 기본 보정 | 무드 보정 | 생성 보완

[ 비교 슬라이더 ]          [ 강도: 자연스럽게 / 스타일 / 생성 ]

생성 보완이 포함된 경우: [ AI 생성 보완 ] [ 적용 내용 보기 ]
```

- 탭 이름과 의미는 고정한다.

  | 트랙 | 사용자 의미 | 처리 |
  |---|---|---|
  | 원본 | 처음 찍은 사진 | 항상 보존 |
  | 기본 보정 | 기울기·밝기·크롭을 자연스럽게 정리 | 온디바이스 |
  | 무드 보정 | 선택한 무드의 색감과 톤 적용 | 온디바이스 |
  | 생성 보완 | 방해 요소 제거·여백 확장처럼 새 픽셀이 필요한 수정 | 서버, `AI 생성 보완` 뱃지 필수 |

- 생성 보완 결과가 없거나 기능이 컷되면 네 번째 탭을 감추고, 빈 상태를 만들지 않는다.
- 생성 후보가 2개 이상이면 사진 영역 안에서 가로 스와이프로 비교한다. 후보를 별도 결과 화면으로 늘리지 않는다.
- 전후 슬라이더, 길게 눌러 원본 보기, 저장·공유, 재생성(가능한 경우), 편집 이력은 이 화면에 모은다.
- 결과를 저장하거나 닫을 때만 1탭 피드백을 5초간 제안한다. 강제로 응답받지 않는다(M14-01).

### 4.6 앨범 — 빠른 재진입을 위한 가벼운 기록

```text
앨범                                         [선택]
[ 전체 ▾ ]

┌──────┐ ┌──────┐ ┌──────┐
│ 사진 │ │ 사진 │ │ 사진 │
│      │ │ ✦    │ │      │
└──────┘ └──────┘ └──────┘
```

- P0은 둥근 모서리의 3열 그리드로 최근 결과를 보여준다.
- `✦`는 생성 보완이 포함된 결과에만 붙인다. 원본과 여러 결과가 연결된 사진은 탭하면 결과 화면으로 연다.
- 일치도, 촬영 가이드 사용 여부, 스타일명, 상세 편집 이력, 즐겨찾기, 레퍼런스 분류는 P1 상세 화면으로 미룬다.
- 로컬 저장소가 원천이며 앱 삭제 시 데이터가 사라질 수 있음을 설정의 개인정보 안내에서 재확인한다.

## 5. 시각·인터랙션 시스템

### 시각 방향

- **배경:** 거의 검은 중성 차콜. 사진·카메라 프리뷰를 가장 밝은 콘텐츠로 만든다.
- **표면:** 검은색보다 한 단계 밝은 차콜. 카드와 바텀시트에만 제한적으로 사용한다.
- **강조색:** 독자적인 저채도 세이지 또는 앰버 한 색을 선택·완료·진행에만 사용한다. 외부 사례의 올리브색·버튼 형태를 그대로 따라 하지 않는다.
- **텍스트:** 따뜻한 흰색을 기본, 중간 회색을 보조로 사용한다. 사진 위 라벨에는 어두운 그라데이션을 넣어 대비를 확보한다.
- **생성 보완:** 일반 성공색과 구분되는 보조색 또는 `AI 생성 보완` 라벨을 사용한다. 반짝임 효과만으로 의미를 전달하지 않는다.

### 카드 규칙

| 컴포넌트 | 규칙 |
|---|---|
| 취향 카드 | 2열·3:4·이미지 중심. 선택은 테두리와 체크로 표시 |
| 무드 카드 | 대표 이미지, 이름 1줄, 행동/분위기 설명 1줄, 탭하면 즉시 카메라 진입 |
| 진단 카드 | 이미지보다 문장 우선. 한 카드에 문제 하나와 제안 하나 |
| 앨범 셀 | 3열 정사각형, 상태 아이콘은 우측 하단에 한 개만 |
| 버튼 | 기본 CTA는 한 화면에 하나. 보조 CTA는 텍스트 또는 외곽선 |

### 문구 규칙

- 기술 측정값을 행동 문장으로 바꾼다.
- 카메라의 실시간 지시는 조사식보다 명령형이 이해하기 쉽다.
- 한 번에 하나의 지시만 쓴다.

| 피해야 할 표현 | 사용할 표현 |
|---|---|
| `Subject scale mismatch: -12%` | `피사체와 조금 더 가까워지세요.` |
| `Horizon roll: 3.2°` | `휴대폰을 조금 오른쪽으로 돌려주세요.` |
| `Outpainting 실행` | `위쪽 여백을 자연스럽게 늘릴게요.` |
| `생성 실패` | `자연스러운 보정만 적용했어요.` |

### 모션 규칙

- 무드 변경: 150~250ms 크로스페이드.
- 목표 위치 도달: 셔터 링 색만 부드럽게 전환.
- 목표 영역 진입: 오버레이 색 전환. 자동 촬영 카운트다운·셔터 링·햅틱은 사용하지 않는다.
- 생성 결과: 결과가 준비되기 전 진행 상태를 보여주고, 준비 후 사진이 갑자기 튀지 않도록 페이드한다.
- 장식적인 반짝임·무한 로딩·과한 카메라 흔들림은 사용하지 않는다.

### 사용성 검증 기준

- 홈/무드에서 카메라까지: **2터치 이하**
- 카메라 진입 후 첫 목표 프레임·안내: **2초 이내**
- 안내 문구 최소 유지: **0.8초**
- 온보딩 시작부터 첫 촬영 화면: **60초 이내**
- 결과 화면에서 원본과 현재 결과 비교: **한 화면에서 가능**
- 네트워크가 없어도 촬영·구도 가이드·기본 보정: **계속 사용 가능**
- 생성 결과가 검증 실패한 경우: **실패한 이미지를 노출하지 않고 기본 보정으로 폴백**

## 8. 기존 문서 개정이 필요한 항목

이 참고서는 구현을 바로 바꾸지 않는다. 다만 4탭 구조를 실제 확정안으로 채택하려면 다음 동기화가 필요하다.

1. 기능명세서 `M15`를 `무드 / 카메라 / 사진 살리기 / 앨범` 구조로 개정한다.
2. `M15-01 홈`의 3대 진입을 무드 탭 안의 CTA로 옮긴다.
3. `M15-02 스타일 탐색`과 `M15-03 스타일 프로필`은 P1 보조 흐름으로 명시한다.
4. 결과 탭은 M13의 4트랙 명칭을 기준으로 유지한다.
5. M3의 프리셋 6종과 무드 화면의 추천 3장을 구분해, 콘텐츠·API·UI 구현자가 서로 다른 수로 오해하지 않게 한다.
6. 기술 스택은 Kotlin/Compose로 확정한다. 이 문서의 화면 설계는 Android 네이티브 구현을 기준으로 한다.

## 9. 최종 체크리스트

- [ ] 화면에 `MOODFRAME`이 사용자용 서비스명으로 노출되지 않는다.
- [ ] 무드 선택은 사진 중심이며, 추천 카드가 3개를 넘지 않는다.
- [ ] 카메라에는 한 번에 한 안내만 표시된다.
- [ ] P0 카메라 캐러셀에는 `무드`, `레퍼런스` 외 카테고리가 없다.
- [ ] 결과는 원본/기본 보정/무드 보정/생성 보완 4트랙으로 설명된다.
- [ ] 생성 기능이 준비되지 않으면 관련 CTA와 빈 탭을 숨긴다.
- [ ] 서버 처리 화면은 업로드 목적과 원본 삭제 사실을 고지한다.
- [ ] 원본은 어떤 화면에서도 삭제·덮어쓰기 되지 않는다.
- [ ] 네트워크, 감지 실패, 저조도, 생성 실패 상태가 각각 정의되어 있다.
- [ ] Day 6 이후 새로운 핵심 화면을 추가하지 않는다.
